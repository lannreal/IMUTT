## 🔮 System Overview

<div align="center">

> **NEXUS AI** is a zero-cost, reverse-engineered gateway to DeepAI's most powerful premium capabilities — built from the ground up with surgical precision.

</div>

**NEXUS AI** (`dp.js`) is an ultra-professional, zero-subscription framework that grants **unrestricted access** to DeepAI's premium infrastructure by reverse-engineering their proprietary cryptographic token system — the `generateIslandKey` algorithm. Every single request is authenticated with a **freshly generated, mathematically valid signature** — making it indistinguishable from a legitimate browser session.

This framework operates seamlessly across **two professional execution environments:**

<div align="center">

| Environment | Description | Command |
|:-----------:|:------------|:-------:|
| 💻 **Interactive CLI** | A sleek, multi-mode terminal interface with animated spinners, typewriter AI output, and model selection menus | `node dp.js` |
| 🌐 **REST API Daemon** | A headless Express.js HTTP server that exposes all functionality as clean, JSON-based REST endpoints — ready for enterprise integration | `node dp.js --api` |

</div>

## 🧠 How It Works — Technical Deep Dive

### 🔑 The `generateIslandKey` Algorithm

The core of NEXUS AI is the reverse-engineered `generateIslandKey` function — a token generator that mimics DeepAI's own client-side JavaScript to produce valid authentication credentials **without any account or subscription.**

```
Step 1 → Generate a random 11-digit numeric seed: myrandomstr
Step 2 → Implement a custom MD5-derivative hash function (myhashfunction)
Step 3 → Hash chain: H(userAgent + H(userAgent + H(userAgent + seed + SALT)))
Step 4 → Format: tryit-{seed}-{triple_chained_hash}
Step 5 → Inject into every request header as: api-key
```

> 🧂 **The Salt:** `hackers_become_a_little_stinkier_every_time_they_hack`
>
> This is DeepAI's own embedded salt string — extracted directly from their minified production JavaScript.

Every API call generates a **brand new, cryptographically valid token** — ensuring no rate-limiting or token reuse detection.

---

## 🌐 API Endpoints Reference

When running in `--api` mode, the daemon exposes the following REST endpoints on `http://localhost:3000`:

---

### `POST /api/generate` — Text-to-Image Generation

Generate high-quality AI images from natural language prompts.

**Request Body:**
```json
{
  "prompt": "A futuristic cyberpunk city at night, neon lights reflecting on wet streets",
  "isQuality": true,
  "style": "hd",
  "negativePrompt": "blurry, low resolution, watermark, ugly"
}
```

| Field | Type | Required | Default | Description |
|:------|:-----|:--------:|:-------:|:------------|
| `prompt` | `string` | ✅ Yes | — | The visual description for image generation |
| `isQuality` | `boolean` | ❌ No | `true` | `true` = HD quality (slower), `false` = Speed mode |
| `style` | `string` | ❌ No | `"hd"` | Image style — `"hd"` or `"gan"` |
| `negativePrompt` | `string` | ❌ No | `null` | Elements to exclude from the generated image |

**Success Response (`200 OK`):**
```json
{
  "status": "success",
  "prompt": "A futuristic cyberpunk city at night...",
  "style": "hd",
  "quality_mode": true,
  "negative_prompt": "blurry, low resolution...",
  "download_url": "https://api.deepai.org/job-view-file/..."
}
```

---

### `POST /api/chat` — Neural Chat Interface

Multi-turn, stateful conversation with any of the 10 supported AI models. Supports Vision AI via `attachment_uuids`.

**Request Body:**
```json
{
  "model": "gemma-4",
  "messages": [
    {
      "role": "user",
      "content": "Describe the architectural patterns shown in this image.",
      "attachment_uuids": ["df99118c-8bdb-4bd9-8bb5-f783e27db59a"]
    }
  ]
}
```

| Field | Type | Required | Default | Description |
|:------|:-----|:--------:|:-------:|:------------|
| `messages` | `array` | ✅ Yes | — | Chat history array with `role` and `content` fields |
| `model` | `string` | ❌ No | `"standard"` | Any model ID from the Supported Models Matrix |
| `attachment_uuids` | `array` | ❌ No | `[]` | UUIDs from pre-uploaded files for Vision AI models |

**Success Response (`200 OK`):**
```json
{
  "status": "success",
  "response": "The image showcases a modular microservices architecture...",
  "history": [
    { "role": "user", "content": "Describe the architectural patterns..." },
    { "role": "assistant", "content": "The image showcases..." }
  ]
}
```

## 🚀 Deployment & Usage

### Prerequisites

Ensure you have **Node.js v16 or higher** installed on your system.

```bash
# Verify your Node.js version
node --version
```

---

### 1️⃣ Install Dependencies

Clone the repository and install the required packages:

```bash
# Clone the repository
git clone https://github.com/lanncodex/nexus-ai.git
cd nexus-ai

# Install all dependencies
npm install axios form-data express cors
```

---

### 2️⃣ Unified Launcher — Interactive Mode

Launch NEXUS AI without any flags to enter the **interactive module selector:**

```bash
node dp.js
```

You will be presented with the following menu:

```
  Modul Tersedia:
   1. Text-to-Image Generator
   2. REST API Server Daemon
   3. Neural Chat Interface

 ▪ Pilih Modul [Def: 1]:
```

---

### 3️⃣ Direct Flag Launch

Skip the menu by using direct execution flags:

```bash
# Launch Text-to-Image Generator CLI
node dp.js --image

# Launch Neural Chat Interface CLI
node dp.js --chat

# Launch REST API Daemon (headless)
node dp.js --api
```

**Flag Reference:**

| Flag | Mode | Description |
|:-----|:----:|:------------|
| *(none)* | Interactive | Displays the module selector menu |
| `--image` or `--cli` | Image CLI | Directly launches the image generation interface |
| `--chat` | Chat CLI | Directly launches the neural chat interface |
| `--api` | REST API | Starts the headless Express.js API daemon on port 3000 |

## 👁️ Vision AI & Image Upload

NEXUS AI supports native binary file upload for Vision-enabled models. Images are uploaded to DeepAI's attachment endpoint and referenced by UUID in subsequent chat messages.

### ✅ Vision-Compatible Models

```
llama-4-scout  |  gemini-2.5-flash-lite  |  gemma-4  |  gpt-5-nano
```

---

### In CLI Mode

Prepend `[image]` followed by the **absolute file path** to your image. Optionally, add your analysis prompt after the path:

```
# Syntax
USER  > [image] "/absolute/path/to/image.png" Your analysis prompt here

# Examples
USER  > [image] "C:\Users\John\Desktop\diagram.png" Explain this system architecture
USER  > [image] "C:\Photos\receipt.jpg" Extract all line items and calculate the total
USER  > [image] "/home/user/chart.png" What trends does this data visualization show?
```

> ⚠️ **Important:** The selected model **must** support Vision. If you send an image to a non-Vision model, the system will display an error and prompt you to select a compatible model.

---

### Via REST API

To use Vision AI through the API, reference the UUID of a pre-uploaded attachment in your chat request:

```json
{
  "model": "gpt-5-nano",
  "messages": [
    {
      "role": "user",
      "content": "What objects are in this image?",
      "attachment_uuids": ["your-uuid-here"]
    }
  ]
}
```
